# WordPress Add-on Specialist

## Purpose

Use this skill when writing content for WordPress plugins, WooCommerce extensions, LearnPress add-ons, LMS extensions, or eCommerce add-ons.

## Source Basis

Consolidated from wordpress-master, technical-writer, business-analyst, and product-manager skills.

## Required WordPress Context

- WordPress version support.
- WooCommerce/LearnPress/platform version support.
- PHP requirements.
- Compatibility notes.
- Admin UX location.
- Frontend UX impact.
- Hooks, filters, REST API, shortcode, block, widget, or template support.
- Data stored and migration concerns.
- Security and permission model.
- Performance implications.

## Content Rules

- Explain technical value in buyer language.
- Mention compatibility only when known.
- Do not claim HPOS, Cart/Checkout Blocks, multisite, multilingual, or payment gateway support unless verified.
- Include implementation notes for technical evaluators without turning marketing copy into developer docs.
- For WooCommerce-style pages, include extension information and quality/compatibility style blocks when data exists.

## Buyer Questions To Answer

- Will it work with my current store/site?
- How hard is setup?
- What happens to existing data?
- What does the admin manage?
- What does the customer/student see?
- What can developers customize?
- What support/docs are available?
