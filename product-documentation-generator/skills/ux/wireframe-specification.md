# Wireframe Specification

## Purpose

Use this skill to create low-fidelity ASCII wireframes that give design and engineering enough structure to begin work.

## Required Screen Format

For each screen include:

- Screen name.
- Purpose.
- Components.
- User actions.
- Navigation.
- Empty states.
- Error states.
- Permission differences if applicable.

## ASCII Wireframe Rules

- Use ASCII layouts only.
- Keep layouts low fidelity.
- Show major regions, controls, form fields, tables, CTAs, alerts, and navigation.
- Do not over-specify colors, spacing, or visual polish unless needed for understanding.

## Example

```text
+------------------------------------------------+
| Header: Product Name                  Settings |
+------------------------------------------------+
| Sidebar        | Main Content                  |
| - Overview     | [Primary CTA]                 |
| - Users        | Table / Cards / Empty State   |
| - Reports      |                               |
+------------------------------------------------+
```
